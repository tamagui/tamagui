## Installation

Toast is not included by default in `tamagui` because Metro would force native apps to install the `burnt` dependency, breaking Expo Go compatibility.

```bash
npm install @tamagui/toast burnt
```

If targeting native, rebuild your React Native app after installing.

<Notice theme="blue">
  For native apps, we recommend [setting up native portals](/docs/components/portal#native-portal-setup-recommended) to preserve React context inside Toast content.
</Notice>

## Anatomy

```tsx
import { toast, Toaster } from '@tamagui/toast'

export default () => (
  <>
    <Toaster />
    {/* Call toast() anywhere in your app */}
  </>
)
```

## Quick Start

```tsx
import { toast, Toaster } from '@tamagui/toast'

function App() {
  return (
    <>
      <Toaster />
      <Button onPress={() => toast('Hello!')}>Show Toast</Button>
    </>
  )
}
```

## API Reference

### toast()

The main function to show toasts. Call it directly or use typed methods.

```tsx
// Basic
toast('Event created')

// With description
toast('Event created', { description: 'Monday, January 3rd at 6:00pm' })

// Typed toasts
toast.success('Saved successfully')
toast.error('Something went wrong')
toast.warning('Please review your input')
toast.info('New update available')

// Loading toast
toast.loading('Uploading...')

// With action button
toast('File deleted', {
  action: {
    label: 'Undo',
    onClick: () => console.log('Undo clicked'),
  },
})

// With cancel button
toast('Confirm action', {
  cancel: {
    label: 'Cancel',
    onClick: () => console.log('Cancelled'),
  },
})

// Dismiss programmatically
const id = toast('Processing...')
toast.dismiss(id)

// Dismiss all
toast.dismiss()
```

### toast.promise()

Show a toast that automatically updates based on a promise's state.

```tsx
toast.promise(saveData(), {
  loading: 'Saving...',
  success: 'Data saved!',
  error: 'Failed to save',
})

// With dynamic messages
toast.promise(fetchUser(id), {
  loading: 'Loading user...',
  success: (data) => `Welcome, ${data.name}!`,
  error: (err) => `Error: ${err.message}`,
  finally: () => console.log('Request completed'),
})

// Unwrap the result
const { unwrap } = toast.promise(asyncOperation(), {
  loading: 'Working...',
  success: 'Done!',
  error: 'Failed',
})

const result = await unwrap()
```

### toast.custom()

Render completely custom JSX for a toast.

```tsx
toast.custom((id) => (
  <YStack padding="$4" backgroundColor="$background">
    <Text>Custom toast content</Text>
    <Button onPress={() => toast.dismiss(id)}>Close</Button>
  </YStack>
))
```

### toast.dismiss()

Dismiss toasts programmatically.

```tsx
// Dismiss specific toast
const id = toast('Hello')
toast.dismiss(id)

// Dismiss all toasts
toast.dismiss()
```

### toast.getToasts() / toast.getHistory()

Access toast state.

```tsx
// Get currently active (non-dismissed) toasts
const activeToasts = toast.getToasts()

// Get all toasts including dismissed ones
const allToasts = toast.getHistory()
```

## Examples

### Position

Control where toasts appear on screen:

```tsx
<Toaster position="top-center" />
<Toaster position="bottom-left" />
```

### Custom Icons

Replace default icons for toast types:

```tsx
import { Check, X, AlertTriangle, Info, Loader } from '@tamagui/lucide-icons'

<Toaster
  icons={{
    success: <Check color="$green10" />,
    error: <X color="$red10" />,
    warning: <AlertTriangle color="$yellow10" />,
    info: <Info color="$blue10" />,
    loading: <Loader color="$gray10" />,
  }}
/>
```

### With Actions

```tsx
toast('Message sent', {
  action: {
    label: 'View',
    onClick: () => navigate('/messages'),
  },
})

// With both action and cancel
toast('Delete this item?', {
  action: {
    label: 'Delete',
    onClick: () => deleteItem(),
  },
  cancel: {
    label: 'Cancel',
    onClick: () => {},
  },
})
```

### Update Existing Toast

Use the same ID to update a toast:

```tsx
const id = toast.loading('Uploading...')

// Later...
toast.success('Upload complete!', { id })
```

### Persistent Toast

Set duration to `Infinity` to keep toast visible until dismissed:

```tsx
toast('Important message', {
  duration: Infinity,
  dismissible: true,
})
```

### Native Toasts

On mobile, toasts can use native system toasts via the [burnt](https://github.com/nandorojo/burnt) package:

```tsx
<Toaster
  burntOptions={{
    from: 'bottom', // iOS only: 'top' | 'bottom'
    preset: 'done', // 'done' | 'error' | 'none'
  }}
/>
```

To disable native toasts and always use custom toasts:

```tsx
<Toaster disableNative />
```

## Types

```tsx
import type {
  ToastT,
  ToastType,
  ToasterPosition,
  ExternalToast,
  PromiseData,
  ToastAction,
  SwipeDirection,
} from '@tamagui/toast'

// ToastType
type ToastType = 'default' | 'success' | 'error' | 'warning' | 'info' | 'loading'

// ToasterPosition
type ToasterPosition =
  | 'top-left'
  | 'top-center'
  | 'top-right'
  | 'bottom-left'
  | 'bottom-center'
  | 'bottom-right'

// SwipeDirection
type SwipeDirection = 'left' | 'right' | 'up' | 'down' | 'horizontal' | 'vertical'
```

## Notes

For Android, you may need to manually re-propagate context when using toasts in modals. This is because React Native doesn't fully support portals yet.

On web, the `Toaster` component automatically renders into a portal at the root level. On native, toasts render inline but are positioned absolutely.
