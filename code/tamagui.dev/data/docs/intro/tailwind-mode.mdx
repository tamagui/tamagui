---
title: Tailwind Mode
description: Tailwind v4 utility syntax for Tamagui components
---

Tailwind mode lets Tamagui components consume Tailwind-style `className` strings
and turn supported utilities into Tamagui styles. It is built for projects that
want Tailwind syntax while keeping Tamagui components, tokens, themes, media, and
pseudo-state handling.

## Configuration

Enable Tailwind-only mode in your Tamagui config:

```tsx
import { createTamagui } from 'tamagui'

const config = createTamagui({
  // ...
  settings: {
    styleMode: 'tailwind',
  },
})
```

Use `styleMode: 'tamagui-and-tailwind'` when you want both classic Tamagui style
props and Tailwind-style `className` utilities.

Set `TAMAGUI_DISABLE_TAILWIND_MODE=1` at build time to keep Tailwind parsing
disabled even when a shared config has `styleMode` set to `tailwind` or
`tamagui-and-tailwind`.

## Basic Syntax

```tsx
import { View } from 'tamagui'

export function Card() {
  return (
    <View className="w-64 bg-white p-4 rounded-lg border hover:bg-gray-100 sm:p-6" />
  )
}
```

Supported utilities are consumed from `className` and emitted as Tamagui styles.
Unknown classes stay in `className`.

| Class | Result |
| --- | --- |
| `w-64` | `width: 256px` |
| `h-20` | `height: 80px` |
| `p-4` | `padding: 16px` |
| `m-2` | `margin: 8px` |
| `gap-3` | `gap: 12px` |
| `rounded-lg` | Tailwind `lg` radius token |
| `border` | `borderWidth: 1px` |
| `border-2` | `borderWidth: 2px` |
| `opacity-50` | `opacity: 0.5` |

## Value Semantics

Numeric spacing and sizing utilities use Tailwind's scale:

```tsx
<View className="p-4 w-24 gap-2" />
// p-4 -> 16px
// w-24 -> 96px
// gap-2 -> 8px
```

Border widths, z-index, opacity, flex values, and raw radius numbers keep their
Tailwind meaning:

```tsx
<View className="border-2 rounded-8 z-10 opacity-50" />
// border-2 -> 2px
// rounded-8 -> 8px
// z-10 -> 10
// opacity-50 -> 0.5
```

Use arbitrary values when you need literal pixels:

```tsx
<View className="w-[100px] p-[10px]" />
```

## Tokens

Class values are resolved against configured Tamagui tokens by name. Do not put
`$` in a class value:

```tsx
<View className="bg-blue5 rounded-lg" />
```

If `$blue5` or `$lg` exists in the active config, the class resolves to that
token. A `$` prefix inside `className` is invalid and the class is preserved:

```tsx
<View className="bg-$blue5" /> // invalid class-mode syntax
```

## Modifiers

Pseudo states, media, platform, and theme modifiers use Tailwind's colon syntax:

```tsx
<View className="bg-white hover:bg-gray-100 press:opacity-80 sm:p-6 dark:bg-black" />
```

Modifiers can be combined:

```tsx
<View className="bg-white sm:hover:bg-blue-500" />
```

## Built-In Mappings

| Utility | Tamagui style |
| --- | --- |
| `flex-row`, `flex-col` | `flexDirection` |
| `flex-wrap`, `flex-nowrap` | `flexWrap` |
| `flex-1`, `flex-auto`, `flex-none` | `flexGrow`, `flexShrink`, `flexBasis` |
| `hidden` | `display: 'none'` |
| `relative`, `absolute` | `position` |
| `items-center`, `items-start` | `alignItems` |
| `justify-between`, `justify-center` | `justifyContent` |
| `w-full`, `w-1/2`, `h-screen` | sizing keywords and fractions |

Configured shorthands are also available. The v6 shorthands include mappings
such as `bg`, `p`, `pt`, `px`, `m`, `mx`, `rounded`, `w`, `h`, `items`,
`justify`, `grow`, `shrink`, and `z`.

## Current Gaps

The conformance harness tracks Tailwind v4 visual parity in
`code/comparisons/conformance`. The current known conversion gaps are:

- single-side border utilities such as `border-l-2`
- corner radius utilities such as `rounded-tl-lg`
- axis inset utilities such as `inset-x-4`
- bare `grow` and `shrink`

Those are planned through multi-target shorthands so the parser stays config
driven rather than accumulating Tailwind-only branches.
