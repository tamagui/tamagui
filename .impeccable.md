## Design Context

### Users

Tamagui contributors and maintainers working in a native desktop development environment. They use this playground to quickly verify that Tamagui components, interaction states, and especially Portal-backed overlays behave correctly on macOS and other desktop targets.

### Brand Personality

Polished, colorful, and technically confident. The interface should feel recognizably Tamagui: expressive enough to make testing enjoyable, but clear and dependable enough to expose regressions without visual ambiguity. It should evoke confidence that native desktop behavior is being tested intentionally.

### Aesthetic Direction

Match the public Tamagui brand with a bright, refined, color-forward interface and support both light and dark themes where practical. Prefer an authored developer-tool composition over a generic dashboard, a literal Tamagotchi theme, or a plain component gallery. Avoid generic AI gradients, excessive cards, glassmorphism, and decorative effects that obscure component boundaries or overlay stacking.

### Design Principles

1. Make test intent visible: every specimen should state what interaction or Portal behavior it validates.
2. Preserve authentic component behavior: use real Tamagui components and providers rather than visual imitations.
3. Design for fast scanning: group related controls, keep state feedback close to the trigger, and make pass/fail observations easy to spot.
4. Exercise desktop-specific behavior deliberately: mouse, keyboard focus, dismissal, positioning, clipping, nesting, and z-order should all have explicit test surfaces.
5. Use expressive color and hierarchy with restraint so overlays remain visually distinguishable from their host content in both themes.
